# Deployment

Two supported deployment targets:

1. **GitHub Codespaces** (recommended for daily use).
2. **Local dev** (Ubuntu / WSL / macOS — Linux containers).

Both share the same `.devcontainer/devcontainer.json`.

---

## 1. Codespaces

### Prerequisites

- GitHub account with Codespaces enabled.
- A free NVIDIA NIM key from [build.nvidia.com](https://build.nvidia.com).
- A Telegram bot token from [@BotFather](https://t.me/BotFather).
- Your own Telegram user id (DM [@userinfobot](https://t.me/userinfobot)).

### Steps

1. **Open in Codespaces.** Repo page → *Code* → *Codespaces* →
   *Create codespace on main*.

2. **Add secrets.** GitHub repo *Settings* → *Secrets and variables* →
   *Codespaces*. Add:

   | Secret                  | Value                              |
   | ----------------------- | ---------------------------------- |
   | `NVIDIA_API_KEY`        | `nvapi-...`                        |
   | `TELEGRAM_BOT_TOKEN`    | `0000000000:AA...`                 |
   | `TELEGRAM_CHAT_ID`      | your numeric user id               |
   | `ALLOWED_TG_USER_IDS`   | comma-separated list of allowed ids |

3. **Rebuild the container.** Command palette →
   *Codespaces: Rebuild Container*. This runs `postCreateCommand` which
   installs `pnpm` and `openclaw`.

4. **Onboard OpenClaw.**

   ```bash
   openclaw onboard
   ```

   When prompted, choose NVIDIA NIM and paste the key. The resulting
   `~/.openclaw/openclaw.json` already includes `allowedUsers` if you set
   that secret.

5. **Start the gateway** (in a persistent tmux pane):

   ```bash
   tmux new -s openclaw
   openclaw gateway
   #   Ctrl+B then D  to detach
   #   tmux attach -t openclaw  to come back
   ```

6. **Smoke test.**

   ```bash
   python -m pytest -q
   ```

7. **Try a skill from the CLI:**

   ```bash
   python skills/research/research.py "what's the state of speculative decoding in late 2025"
   python skills/review/review.py        # reviews current diff
   ```

8. **Telegram check.** DM the bot — it should respond. If it doesn't,
   verify `ALLOWED_TG_USER_IDS` includes your numeric id.

## 2. Local dev (Ubuntu / WSL / macOS)

```bash
git clone https://github.com/eyasir329/openclaw-lab.git
cd openclaw-lab
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
npm install -g openclaw
cp .env.example .env  # then edit .env with real values
```

Set environment variables either by `source .env` + `export` lines, or by
your favourite tooling (`direnv`, `mise`, `taskfile`).

Run the gateway and skills the same way as in Codespaces.

## Configuration reference

### Environment variables

See [`.env.example`](../.env.example) for the canonical list. The hot ones:

| Variable                | Required | Description                                  |
| ----------------------- | -------- | -------------------------------------------- |
| `NVIDIA_API_KEY`        | yes      | NIM API key.                                 |
| `TELEGRAM_BOT_TOKEN`    | when bot is used | Bot token from @BotFather.            |
| `TELEGRAM_CHAT_ID`      | when bot is used | Numeric chat id to send to.           |
| `ALLOWED_TG_USER_IDS`   | yes when bot is exposed | Comma-separated allowlist. **Fail-closed.** |
| `CAVEMAN_MODEL`         | no       | Override default compression model.          |
| `OPENCLAW_AUDIT_LOG`    | no       | Custom path for the audit log.               |
| `CAVEMAN_DEFAULT_MODE`  | no       | `off | lite | full | ultra | wenyan-*`       |

### `~/.openclaw/openclaw.json`

Generated by `openclaw onboard`. Key sections:

```json
{
  "models": {
    "providers": {
      "nvidia": {
        "baseUrl": "https://integrate.api.nvidia.com/v1",
        "api": "openai-completions",
        "timeoutSeconds": 300
      }
    }
  },
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "0000000000:...",
      "chatId": "111111111",
      "allowedUsers": [111111111]
    }
  }
}
```

`timeoutSeconds: 300` is required — some NIM models have ~140 s cold-start
latency. Lower values cause spurious failures.

## Operational tasks

### Tail the audit log

```bash
tail -f logs/audit.jsonl | jq -c '.'
```

### Check ensemble health

```bash
python - <<'EOF'
import asyncio, sys
sys.path.insert(0, '.')
from core.config import load_runtime_config
from core.nim_client import call_nim
import aiohttp

async def go():
    rc = load_runtime_config()
    async with aiohttp.ClientSession() as s:
        r = await call_nim(s, rc.nim_api_key, "moonshotai/kimi-k2.6",
                           system=None, user="Reply with 'pong'.",
                           max_tokens=10, timeout=30)
        print(r)

asyncio.run(go())
EOF
```

### Run only one ensemble model

Edit the `models=` kwarg passed to `run_ensemble`, e.g.:

```python
result = await run_ensemble(
    session, nim_key,
    scorer_system=..., scorer_user=..., fusion_system=...,
    models=[{"id": "moonshotai/kimi-k2.6", "label": "kimi-only", "tier": "S",
             "weight": 1.0, "timeout": 180}],
)
```

The orchestrator detects N=1 and skips the fusion call.

### Rotate `NVIDIA_API_KEY`

1. Generate a new key at [build.nvidia.com](https://build.nvidia.com).
2. Update the Codespaces secret OR `.env` file.
3. Rebuild the container (`Codespaces: Rebuild Container`) — env vars are
   set at startup.
4. Optionally revoke the old key on the same page.

### Disable a skill

Delete or rename the skill directory. Slash commands that reference it
will show "skill not found" but nothing else breaks. The remaining skills
continue working.

## Troubleshooting

| Symptom                                        | Most-likely fix                                                    |
| ---------------------------------------------- | ------------------------------------------------------------------ |
| `NVIDIA_API_KEY is not set`                    | Set the Codespaces secret + rebuild container, or `export` it.     |
| Gateway prints "permission denied" on port     | Another `openclaw` process is running. `pkill -f "openclaw gate"`. |
| Telegram bot silent                            | `ALLOWED_TG_USER_IDS` does not include your id (fail-closed).      |
| 6/6 ensemble models fail                       | Network outage, expired key, or NIM degraded. Re-run; check `openclaw status`. |
| `gitleaks` flags a placeholder                 | Add the literal to `.gitleaks.toml` allowlist.                     |
| Tests pass locally but fail in CI              | `NVIDIA_API_KEY` is unset in CI on purpose — tests must monkeypatch. |
